# Project Guidelines

## Context Tracking
- Check CONTEXT.md at the repo root before responding when it exists.
- If CONTEXT.md is missing, create it at the repo root before finishing the task.
- After completing any prompt, update CONTEXT.md with durable context, current task state, and notable decisions.
- Keep CONTEXT.md short, factual, and easy to scan.

## Workspace Scope
- This is a monorepo. Keep changes localized to the relevant app or package.
- Prefer existing scripts and conventions in the area being edited.
